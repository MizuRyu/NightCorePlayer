## tasks
エージェントがそれぞれ独立してタスクを実行できるように、タスク定義と成果物をまとめる箇所。
ここでは、タスク定義を参照し、このリポジトリ内の主に docs/ にあるようなドキュメントを参照し、仕様を固めることや、
実装計画を立てる場として利用する。
ここで作成する計画はドラフト版であり、ユーザーレビューで問題点がないことが確認された後、`docs/spec/`に正式な仕様として昇格される予定。

それぞれの`tasksNN/`には、`todo.txt` のような形でユーザーの要望が記載されている。

**重要: 曖昧な仕様のまま進めず、ユーザーに不明点は適宜質問するようにしてください。基本的に認識がズレて進むことは避けたいです。ユーザーが決めきれない場合は、仕様としていくつかの選択肢を提示してあげて下さい**

## sturucture
```
tasks/
├── tasks01 # タスク単位。それぞれ割り当てられたタスクの中間成果物やドラフトの仕様、実装計画などをまとめる。
├── tasksNN
├── README.md  # タスク定義と成果物のまとめ

## 注意点
- 可能な限り、spec.md（仕様） implementation-plan.md(実装計画) research_xxx.md（調査結果）などのドキュメントを分けて、成果物を整理してください。
- `data-structure.md`のような、タスクごとに必要なドキュメントを作成することも可能です。上記はあくまで一例に過ぎないです。大切なのは関心事でドキュメントを分けて記載するニュアンスを伝えたいのです。

## 補足
- 検索が必要な場合は、適宜web検索を行い情報を収集してください。
- あなたには、`Microsoft Agent Framework` に関する便利ツールが与えられているのでこれらも有効活用してください。
